# PR 提交计划

根据作品要求，建议从 2026-06-12 到 2026-06-14 期间保持多个小 PR，避免最后一天一次性导入代码。

## PR 1：项目初始化与基础界面

- 功能描述：搭建静态 Web 应用，创建画布、状态栏、识别文本区和执行日志。
- 实现思路：使用 HTML、CSS 和 Canvas 2D API，无第三方依赖。
- 测试方式：本地启动静态服务，确认页面可访问，画布显示正常。

## PR 2：语音识别接入

- 功能描述：接入 Web Speech API，支持普通话连续识别并展示实时文本。
- 实现思路：封装浏览器 SpeechRecognition，处理 start、end、error、result 事件。
- 测试方式：允许麦克风权限，说一句指令，确认识别文本显示。

## PR 3：绘图指令解析与基础图形

- 功能描述：支持颜色、位置、大小、数量和基础形状绘制。
- 实现思路：把语音文本解析为 `{ shape, color, position, size, count }`，再调用 Canvas 绘制。
- 测试方式：分别测试圆形、矩形、三角形、线条、星星和爱心。

## PR 4：复杂指令拆解与场景元素

- 功能描述：支持太阳、云、树、房子、花朵、山峰，并支持一条语音拆成多个动作。
- 实现思路：按连接词拆分语句，逐条执行绘图动作。
- 测试方式：说“画一个太阳在左上角，然后画三朵云在上方”。

## PR 5：画布编辑、导出与文档

- 功能描述：支持撤销、重做、清空、导出，并补齐 README、设计文档和 demo 脚本。
- 实现思路：使用 Canvas ImageData 保存历史快照，使用 `toDataURL` 导出 PNG。
- 测试方式：执行撤销、重做、清空、导出，确认 README 中记录依赖和演示说明。

## 注意事项

- 每个 PR 只做一件事。
- PR 标题一句话说明改动。
- PR 描述必须包含功能描述、实现思路和测试方式。
- 引用浏览器 Web Speech API 和 Canvas API 时，在 README 中说明依赖。
- 如果复用任何旧代码，必须在 PR 描述中说明来源。
