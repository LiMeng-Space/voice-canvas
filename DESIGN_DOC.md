# Voice Canvas 设计文档

## 最终提交信息

- 作品名称：Voice Canvas - AI 语音绘图工具
- 代码仓库：https://github.com/LiMeng-Space/voice-canvas
- Demo 视频：https://www.bilibili.com/video/BV1TwJA6MEc5/
- 运行方式：使用 Chrome 或 Edge 打开项目，本地执行 `python -m http.server 5173` 后访问 `http://localhost:5173`
- 技术栈：HTML、CSS、JavaScript、Canvas 2D API、Web Speech API、Web Speech Synthesis API
- 安全说明：项目无后端服务、无第三方云服务、无 API Key，不包含账号密码或私密凭据

本文档为最终评审版本，说明作品目标、已实现能力、技术架构、持续迭代记录和验收方式。

## 1. 作品目标

开发一款纯语音控制的绘图工具，用户不使用鼠标或键盘完成绘图创作。作品重点展示语音指令理解、复杂指令拆解、响应延迟控制和错误容忍能力。

## 2. 用户流程

1. 用户打开页面并授权麦克风。
2. 用户说出自然语言绘图指令。
3. 系统实时展示识别文本。
4. 系统将复杂语句拆成多个绘图动作。
5. 每个动作立即渲染到 Canvas。
6. 用户继续用语音修改、删除、撤销、重做、清空或导出作品。

## 3. 计划支持的指令能力

| 能力 | 计划说明 |
| --- | --- |
| 基础图形 | 圆形、矩形、三角形、线条、星星、爱心 |
| 可配置人物 | 性别/年龄段、发型、眼镜、高矮、体型、肤色、上衣、下装、鞋子类型、帽子类型 |
| 场景元素 | 太阳、云、树、房子、花朵、山峰 |
| 运动场景模板 | 通过“我喜欢打羽毛球”等口语表达生成球场、球网、人物、球拍和羽毛球 |
| 属性控制 | 颜色、大小、数量、位置、填充、描边、线宽 |
| 画布控制 | 背景、撤销、重做、清空、导出 |
| 文字绘制 | 在指定位置写入语音中的文本 |
| 演示模式 | 通过“播放演示”语音指令自动生成完整示例场景 |
| 执行反馈 | 展示绘图步数、撤销/重做状态、最近结果和执行日志 |
| 局部修改与删除 | 修改最近人物的服装/鞋帽属性，删除最近一个绘图元素 |
| 目标选择 | 按元素类型或画面位置选择要修改/删除的目标 |
| 语音识别修正 | 将常见同音/近音识别错误自动归一到标准指令词 |
| 复杂指令拆解 | 按“然后、再、同时、和”等连接词拆成动作队列 |
| 容错处理 | 支持常见同义词、中文数字、模糊表达和未识别提示 |
| 语音反馈 | 操作成功后用语音播报执行结果 |

## 4. 已实现能力

- 基于 Web Speech API 的普通话连续识别。
- 基于 Canvas 2D 的图形渲染。
- 支持 13 类图形和场景元素。
- 支持可配置人物绘制，可组合性别、年龄段、发型、眼镜、高矮、体型、肤色、上衣、下装、鞋子类型和帽子类型。
- 支持羽毛球运动场景模板，可通过“我喜欢打羽毛球”“画我在球场打球”等口语表达生成组合画面。
- 支持常见颜色、数量、大小、位置、填充和描边。
- 支持“画一个黄色太阳在左上角，然后画三朵蓝色云在上方”这类复杂指令拆解。
- 支持撤销、重做、清空、导出 PNG。
- 支持执行日志、实时识别文本和当前画笔状态展示。
- 支持绘图步数、撤销/重做状态和最近执行结果展示。
- 支持 Web Speech Synthesis 语音反馈。
- 增强口语容错，支持“生成、创建、放置、写上、底色、顶部、底部、正中、十一朵”等常见自然表达。
- 支持“播放演示”语音指令，一次性生成可用于 demo 视频的完整绘图场景。
- 支持最近元素的局部修改与删除，可修改最近人物的上衣、下装、鞋子、帽子，也可删除最近一个元素。
- 支持按类型和位置选择元素，例如“删除右上角的太阳”“把左边的人上衣改成白色”。
- 支持语音识别自动修正，按动作、颜色、图形、方位、服装鞋帽等类别将“播放眼视”“黄涩泰阳”“云多”“戴眼睛”“羽毛求”“身分”“下发”等归一到标准指令词。

## 5. 未完成能力与原因

| 未完成项 | 原因说明 |
| --- | --- |
| 任意自由曲线绘制 | 72 小时作品周期内优先保证稳定的语音到结构化绘图；自由曲线需要更复杂的路径规划和语音连续控制 |
| 云端大模型语义理解 | 为降低部署成本和隐私风险，当前版本采用本地规则解析；后续可接入大模型提升开放指令理解 |
| 多轮追问纠错 | 当前只做未识别提示，尚未实现“你是想画圆还是太阳”这类澄清对话 |
| 完全免点击启动麦克风 | 浏览器安全策略通常要求用户通过一次手势授权麦克风；作品保证授权后的绘图操作纯语音完成 |

## 6. 架构设计

```text
语音输入
  ↓
Web Speech API
  ↓
实时识别文本展示
  ↓
语音识别修正 applySpeechCorrections / speechCorrectionRules
  ↓
文本标准化 normalizeText
  ↓
复杂指令拆解 splitCommands
  ↓
意图识别 executeCommand
  ↓
参数解析 parseOptions
  ↓
绘图/编辑/删除分发
  ├─ 基础图形 drawShape
  ├─ 人物绘制 drawPerson
  ├─ 场景模板 playBadmintonScene / playDemoScene
  ├─ 文字绘制 drawText
  ├─ 目标查找 findTargetElementIndex
  ├─ 局部修改 editRecentPerson
  └─ 元素删除 deleteRecentElement
  ↓
元素状态 state.elements / baseImageData
  ↓
重新渲染 renderElementsFromBase
  ↓
Canvas 画布展示
  ↓
历史快照 saveSnapshot
```

核心文件职责：

- `index.html`：页面结构，包含画布、麦克风按钮、实时识别区、设置面板和执行反馈面板。
- `styles.css`：页面布局、响应式样式、画布区域和侧栏状态面板样式。
- `app.js`：项目主代码，包含语音识别、指令纠错、指令拆解、参数解析、Canvas 绘图、人物绘制、场景模板、局部修改、目标删除、历史快照和导出逻辑。
- `README.md`：评审入口、运行方式、Demo 视频、支持指令、依赖说明和安全说明。
- `DESIGN_DOC.md`：作品设计、架构、能力边界、迭代记录和验收方式。

元素状态设计：

- 对可复绘元素记录类型、位置、颜色、数量、人物属性等结构化信息。
- 对背景和基础画布内容保存 `baseImageData`，用于删除或修改元素后重绘画面。
- 修改或删除元素时，先通过类型、位置或最近元素定位目标，再更新 `state.elements`，最后重新渲染。
- 每次有效绘图、修改、删除或画布操作后保存历史快照，保证撤销和重做可用。

## 7. 响应延迟控制

- 使用浏览器原生 Web Speech API，避免额外网络请求。
- 在最终识别结果出现后立即执行绘图。
- 复杂指令拆成同步动作队列，减少等待感。
- Canvas 使用固定 1280 x 720 坐标系，避免频繁布局计算。

- 导出的 PNG 能打开并包含当前画布内容。
