# Voice Canvas 设计文档

## 1. 作品目标

开发一款纯语音控制的绘图工具，用户不使用鼠标或键盘完成绘图创作。作品重点展示语音指令理解、复杂指令拆解、响应延迟控制和错误容忍能力。

## 2. 用户流程

1. 用户打开页面并授权麦克风。
2. 用户说出自然语言绘图指令。
3. 系统实时展示识别文本。
4. 系统将复杂语句拆成多个绘图动作。
5. 每个动作立即渲染到 Canvas。
6. 用户继续用语音撤销、重做、清空或导出作品。

## 3. 计划支持的指令能力

| 能力 | 计划说明 |
| --- | --- |
| 基础图形 | 圆形、矩形、三角形、线条、星星、爱心 |
| 场景元素 | 太阳、云、树、房子、花朵、山峰 |
| 属性控制 | 颜色、大小、数量、位置、填充、描边、线宽 |
| 画布控制 | 背景、撤销、重做、清空、导出 |
| 文字绘制 | 在指定位置写入语音中的文本 |
| 复杂指令拆解 | 按“然后、再、同时、和”等连接词拆成动作队列 |
| 容错处理 | 支持常见同义词、中文数字、模糊表达和未识别提示 |
| 语音反馈 | 操作成功后用语音播报执行结果 |

## 4. 已实现能力

- 基于 Web Speech API 的普通话连续识别。
- 基于 Canvas 2D 的图形渲染。
- 支持 13 类图形和场景元素。
- 支持常见颜色、数量、大小、位置、填充和描边。
- 支持“画一个黄色太阳在左上角，然后画三朵蓝色云在上方”这类复杂指令拆解。
- 支持撤销、重做、清空、导出 PNG。
- 支持执行日志、实时识别文本和当前画笔状态展示。
- 支持 Web Speech Synthesis 语音反馈。

## 5. 未完成能力与原因

| 未完成项 | 原因说明 |
| --- | --- |
| 任意自由曲线绘制 | 72 小时作品周期内优先保证稳定的语音到结构化绘图；自由曲线需要更复杂的路径规划和语音连续控制 |
| 云端大模型语义理解 | 为降低部署成本和隐私风险，当前版本采用本地规则解析；后续可接入大模型提升开放指令理解 |
| 多轮追问纠错 | 当前只做未识别提示，尚未实现“你是想画圆还是太阳”这类澄清对话 |
| 完全免点击启动麦克风 | 浏览器安全策略通常要求用户通过一次手势授权麦克风；作品保证授权后的绘图操作纯语音完成 |

## 6. 架构设计

```text
语音输入
  ↓
Web Speech API
  ↓
文本标准化 normalizeText
  ↓
复杂指令拆解 splitCommands
  ↓
意图识别 executeCommand
  ↓
参数解析 parseOptions
  ↓
Canvas 绘图 drawShape / drawText
  ↓
历史快照 saveSnapshot
```

## 7. 响应延迟控制

- 使用浏览器原生 Web Speech API，避免额外网络请求。
- 在最终识别结果出现后立即执行绘图。
- 复杂指令拆成同步动作队列，减少等待感。
- Canvas 使用固定 1280 x 720 坐标系，避免频繁布局计算。

## 8. 测试方式

手动测试以下语音指令：

```text
画一个红色太阳在左上角
画三朵蓝色云在上方
画两座绿色山在下方
写文字 夏日实训 在中间
把背景改成浅蓝色
撤销
重做
清空画布
导出作品
```

验收标准：

- 指令可被识别并记录到执行日志。
- 图形位置和颜色与指令基本一致。
- 撤销和重做恢复正确。
- 导出的 PNG 能打开并包含当前画布内容。
